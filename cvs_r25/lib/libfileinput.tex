\section{\module{fileinput} ---
%         Iterate over lines from multiple input streams}
	  複数の入力ストリームをまたいだ行の繰り返し処理をサポートする。}
\declaremodule{standard}{fileinput}
\moduleauthor{Guido van Rossum}{guido@python.org}
\sectionauthor{Fred L. Drake, Jr.}{fdrake@acm.org}

\modulesynopsis{
Perlのような複数の入力ストリームをまたいだ行の繰り返し処理をサポートする
(その場で保存する機能つき)。}


このモジュールは標準入力やファイルの並びにまたがるループを素早く書くため
のヘルパークラスと関数を提供しています。

典型的な使い方は以下の通りです:

\begin{verbatim}
import fileinput
for line in fileinput.input():
    process(line)
\end{verbatim}


このプログラムは\code{sys.argv[1:]}に含まれる全てのファイルをまたいで繰
り返します。もし該当するものがなければ、\code{sys.stdin}がデフォルトとし
て扱われます。ファイル名として\code{'-'}が与えられた場合も、
\code{sys.stdin}に置き換えられます。別のファイル名リストを使いたい時には、
\function{input()}の最初の引数にリストを与えます。単一ファイル名の文字列
も受け付けます。

全てのファイルはデフォルトでテキストモードでオープンされます。しかし、
\function{input()} や \class{FileInput()} をコールする際に \var{mode}
パラメータを指定すれば、これをオーバーライドすることができます。
オープン中あるいは読み込み中にI/Oエラーが発生した場合には、
\exception{IOError}が発生します。

\code{sys.stdin}が2回以上使われた場合は、2回目以降は行を返しません。
ただしインタラクティブに利用している時や明示的にリセット
(\code{sys.stdin.seek(0))}を使う)を行った場合はその限りではありません。

空のファイルは開いた後すぐ閉じられます。
空のファイルはファイル名リストの最後にある場合にしか外部に影響を与えませ
ん。

ファイルの最後が改行文字で終っていない場合には、最後に改行文字を追加して
返します。

ファイルのオープン方法を制御するためのオープン時フックは、
\function{input()} あるいは \class{FileInput()} の \var{openhook}
パラメータで設定します。このフックは、ふたつの引数 \var{filename}
と \var{mode} をとる関数でなければなりません。そしてその関数の返り値は
オープンしたファイルオブジェクトとなります。このモジュールには、便利な
フックが既に用意されています。

以下の関数がこのモジュールの基本的なインタフェースです:

\begin{funcdesc}{input}{\optional{files\optional{, inplace\optional{,
                        backup\optional{, mode\optional{, openhook}}}}}}

\class{FileInput}クラスのインスタンスを作ります。生成されたインスタンス
は、このモジュールの関数群が利用するグローバルな状態として利用されます。
この関数への引数は\class{FileInput}クラスのコンストラクタへ渡されます。

  \versionchanged[パラメータ \var{mode} および \var{openhook} が追加さ
  れました]{2.5}
\end{funcdesc}

以下の関数は\function{input()}関数によって作られたグローバルな状態を利用
します。アクティブな状態が無い場合には、\exception{RuntimeError}が発生し
ます。

\begin{funcdesc}{filename}{}

現在読み込み中のファイル名を返します。一行目が読み込まれる前は
 \code{None}を返します。
\end{funcdesc}

\begin{funcdesc}{fileno}{}
  現在のファイルの ``ファイルデスクリプタ'' を整数値で返します。ファイ
  ルがオープンされていない場合 (最初の行の前、ファイルとファイルの間)
  は \code{-1} を返します。
\versionadded{2.5}
\end{funcdesc}

\begin{funcdesc}{lineno}{}
最後に読み込まれた行の、累積した行番号を返します。1行目が読み込まれる前
 は\code{0}を返します。最後のファイルの最終行が読み込まれた後には、その
 行の行番号を返します。
\end{funcdesc}

\begin{funcdesc}{filelineno}{}

現在のファイル中での行番号を返します。1行目が読み込まれる前は\code{0}を
 返します。最後のファイルの最終行が読み込まれた後には、その
 行のファイル中での行番号を返します。

\end{funcdesc}

\begin{funcdesc}{isfirstline}{}
最後に読み込まれた行がファイルの1行目ならTrue、そうでなければFalseを返します。 

\end{funcdesc}

\begin{funcdesc}{isstdin}{}
最後に読み込まれた行が\code{sys.stdin}から読まれていればTrue、そうでなけ
 ればFalseを返します。 
\end{funcdesc}

\begin{funcdesc}{nextfile}{}

現在のファイルを閉じます。次の繰り返しでは(存在すれば)次のファイルの最初
 の行が読み込まれます。閉じたファイルの読み込まれなかった行は、累積の行
 数にカウントされません。ファイル名は次のファイルの最初の行が読み込まれ
 るまで変更されません。最初の行の読み込みが行われるまでは、この関数は呼
 び出されても何もしませんので、最初のファイルをスキップするために利用す
 ることはできません。最後のファイルの最終行が読み込まれた後にも、この関
 数は呼び出されても何もしません。

\end{funcdesc}

\begin{funcdesc}{close}{}
シーケンスを閉じます。
\end{funcdesc}


このモジュールのシーケンスの振舞いを実装しているクラスの
サブクラスを作ることもできます。

\begin{classdesc}{FileInput}{\optional{files\optional{,
                             inplace\optional{, backup\optional{,
                             mode\optional{, openhook}}}}}}
\class{FileInput}クラスはモジュールの関数に対応するメソッド
\method{filename()}、\method{fileno()}、\method{lineno()}、
\method{fileline()}、\method{isfirstline()}、\method{isstdin()}、
\method{nextfile()} および \method{close()}
を実装しています。それに加えて、次の入力行を返す
 \method{readline()}メソッドと、シーケンスの振舞
 いの実装をしている\method{__getitem__()}メソッドがあります。
シーケンスはシーケンシャルに読み込むことしかできません。つまりランダムア
 クセスと\method{readline()}を混在させることはできません。

  \var{mode} を使用すると、\function{open()} に渡すファイルモードを指
  定することができます。これは \code{'r'}、\code{'rU'}、\code{'U'} お
  よび \code{'rb'} のうちのいずれかとなります。

  \var{openhook} を指定する場合は、ふたつの引数 \var{filename} と
  \var{mode} をとる関数でなければなりません。この関数の返り値は、オー
  プンしたファイルオブジェクトとなります。\var{inplace} と
  \var{openhook} を同時に使うことはできません。

  \versionchanged[パラメータ \var{mode} および \var{openhook} が追加さ
  れました]{2.5}
\end{classdesc}


\strong{その場で保存するオプション機能:} 

キーワード引数 \code{\var{inplace}=1} が \function{input()}か
\class{FileInput}クラスのコンストラクタに渡された場合には、
入力ファイルはバックアップファイルに移動され、標準出力が入力ファイルに設
定されます(バックアップファイルと同じ名前のファイルが既に存在していた
場合には、警告無しに置き替えられます)。
これによって入力ファイルをその場で書き替えるフィルタを書くことができます。
キーワード引数\code{\var{backup}='.<拡張子>'}も与えられていれば、バック
アップファイルの拡張子を決めることができます。デフォルトでは
\code{'.bak'}になっています。出力先のファイルが閉じられればバックアップ
ファイルは消されます。その場で保存する機能は、標準入力を読み込んでいる間
は無効にされます。


\strong{警告:} 現在の実装はMS-DOSの8+3ファイルシステムでは動作しません。


このモジュールには、次のふたつのオープン時フックが用意されています。

\begin{funcdesc}{hook_compressed}{filename, mode}
  gzipやbzip2で圧縮された(拡張子が\code{'.gz'}や\code{'.bz2'}の)
  ファイルを、\module{gzip}モジュールや\module{bz2}モジュールを使って
  透過的にオープンします。ファイルの拡張子が\code{'.gz'}や
  \code{'.bz2'}でない場合は、通常通りファイルをオープンします
  (つまり、\function{open()} をコールする際に伸長を行いません)。

  使用例: 
  \samp{fi = fileinput.FileInput(openhook=fileinput.hook_compressed)}

  \versionadded{2.5}
\end{funcdesc}

\begin{funcdesc}{hook_encoded}{encoding}
  各ファイルを\function{codecs.open()}でオープンするフックを返します。
  指定した\var{encoding}でファイルを読み込みます。

  使用例:
  \samp{fi = fileinput.FileInput(openhook=fileinput.hook_encoded("iso-8859-1"))}

  \note{このフックでは、指定した\var{encoding}によっては
    \class{FileInput}がUnicode文字列を返す可能性があります。}
  \versionadded{2.5}
\end{funcdesc}
