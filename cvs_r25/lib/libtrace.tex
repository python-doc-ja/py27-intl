\section{\module{trace} ---
         Python ステートメント実行のトレースと追跡}

\declaremodule{standard}{trace}
\modulesynopsis{Python ステートメント実行のトレースと追跡}

\module{trace} モジュールはプログラム実行のトレースを可能にし, generate
ステートメントのカバレッジリストを注釈付きで生成して、呼び出し元/呼び出し先の
関連やプログラム実行中に実行された関数のリストを出力します。これは別個の
プログラム中またはコマンドラインから利用することができます。

\subsection{コマンドラインからの利用\label{trace-cli}}

\module{trace} モジュールはコマンドラインから起動することができます。これは次のように単純です。

\begin{verbatim}
python -m trace --count somefile.py ...
\end{verbatim}

これで、\file{somefile.py} の実行中に import された Python モジュールの注釈付きリストが生成されます。

以下のコマンドライン引数がサポートされています：

\begin{description}
\item[\longprogramopt{trace}, \programopt{-t}]
実行されるままに行を表示します。

\item[\longprogramopt{count}, \programopt{-c}]
プログラム完了時に、それぞれのステートメントが何回実行されたかを示す
注釈付きリストのファイルを生成します。

\item[\longprogramopt{report}, \programopt{-r}]
\longprogramopt{count} と \longprogramopt{file} 引数を使った、過去の
プログラム実行結果から注釈付きリストのファイルを生成します。

\item[\longprogramopt{no-report}, \programopt{-R}]
注釈付きリストを生成しません。これは \longprogramopt{count} を何度か走らせてから
最後に単一の注釈付きリストを生成するような場合に便利です。

\item[\longprogramopt{listfuncs}, \programopt{-l}]
プログラム実行の際に実行された関数を列挙します。

\item[\longprogramopt{trackcalls}, \programopt{-T}]
プログラム実行によって明らかになった呼び出しの関連を生成します。

\item[\longprogramopt{file}, \programopt{-f}]
カウント(count) を含む（べき）ファイルに名前をつけます。

\item[\longprogramopt{coverdir}, \programopt{-C}]
中に注釈付きリストのファイルを保存するディレクトリを指定します。

\item[\longprogramopt{missing}, \programopt{-m}]
注釈付きリストの生成時に、実行されなかった行に `\code{>>>>>>}' の印を付けます。

\item[\longprogramopt{summary}, \programopt{-s}]
\longprogramopt{count} または \longprogramopt{report} の利用時に、
処理されたファイルそれぞれの簡潔なサマリを標準出力(stdout)に書き出します。

\item[\longprogramopt{ignore-module}]
指定されたモジュールとそのサブモジュールを（パッケージだった場合に）無視します。
複数回指定できます。

\item[\longprogramopt{ignore-dir}]
指定されたディレクトリとサブディレクトリ中のモジュールとパッケージを
全て無視します。複数回指定できます。
\end{description}

\subsection{プログラミングインターフェース\label{trace-api}}

\begin{classdesc}{Trace}{\optional{count=1\optional{, trace=1\optional{,
                         countfuncs=0\optional{, countcallers=0\optional{,
                         ignoremods=()\optional{, ignoredirs=()\optional{,
                         infile=None\optional{, outfile=None}}}}}}}}}
文(statement)や式(expression)の実行をトレースするオブジェクトを作成します。
全てのパラメタがオプションです。\var{count} は行数を数えます。
\var{trace} は行実行のトレースを行います。\var{countfuncs} は実行中に
呼ばれた関数を列挙します。\var{countcallers} は呼び出しの関連の追跡を
行います。\var{ignoremods} は無視するモジュールやパッケージのリストです。
\var{ignoredirs} は無視するパッケージやモジュールを含むディレクトリのリストです。
\var{infile} は保存された集計(count)情報を読むファイルです。\var{outfile} は
更新された集計(count)情報を書き出すファイルです。
\end{classdesc}

\begin{methoddesc}[Trace]{run}{cmd}
\var{cmd} を、Trace オブジェクトのコントロール下で
現在のトレースパラメタのもとに実行します。
\end{methoddesc}

\begin{methoddesc}[Trace]{runctx}{cmd\optional{, globals=None\optional{,
                                  locals=None}}}
\var{cmd} を、Trace オブジェクトのコントロール下で
現在のトレースパラメタのもと、定義されたグローバルおよびローカル環境で
実行します。定義しない場合、\var{globals} と \var{locals} はデフォルトで
空の辞書となります。
\end{methoddesc}

\begin{methoddesc}[Trace]{runfunc}{func, *args, **kwds}
与えられた引数の \var{func} を、Trace オブジェクトのコントロール下で
現在のトレースパラメタのもとに呼び出します。
\end{methoddesc}

これはこのモジュールの使い方を示す簡単な例です：

\begin{verbatim}
import sys
import trace

# Trace オブジェクトを、無視するもの、トレースや行カウントのいずれか
# または両方を行うか否かを指定して作成します。
tracer = trace.Trace(
    ignoredirs=[sys.prefix, sys.exec_prefix],
    trace=0,
    count=1)

# 与えられたトレーサを使って、コマンドを実行します。
tracer.run('main()')

# 出力先を /tmp としてレポートを作成します。
r = tracer.results()
r.write_results(show_missing=True, coverdir="/tmp")
\end{verbatim}
