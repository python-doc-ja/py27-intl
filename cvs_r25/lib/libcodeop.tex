\section{\module{codeop} ---
         Pythonコードをコンパイルする}

% LaTeXed from excellent doc-string.

\declaremodule{standard}{codeop}
\sectionauthor{Moshe Zadka}{moshez@zadka.site.co.il}
\sectionauthor{Michael Hudson}{mwh@python.net}
\modulesynopsis{(完全ではないかもしれない)Pythonコードをコンパイルする。}

\refmodule{code}モジュールで行われているようなPythonのread-eval-printループをエミュレートするユーティリティを\module{codeop}モジュールは提供します。結果的に、直接モジュールを使いたいとは思わないかもしれません。あなたのプログラムにこのようなループを含めたい場合は、代わりに\refmodule{code}モジュールを使うことをおそらく望むでしょう。

この仕事には二つの部分があります: 

\begin{enumerate}
  \item 入力の一行がPythonの文として完全であるかどうかを見分けられること: 簡単に言えば、次が`\code{>>>~}'か、あるいは`\code{...~}'かどうかを見分けます。
  \item どのfuture文をユーザが入力したのかを覚えていること。したがって、実質的にそれに続く入力をこれらとともにコンパイルすることができます。
\end{enumerate}

\module{codeop}モジュールはこうしたことのそれぞれを行う方法とそれら両方を行う方法を提供します。


前者は実行するには:

\begin{funcdesc}{compile_command}
                {source\optional{, filename\optional{, symbol}}}
Pythonコードの文字列であるべき\var{source}をコンパイルしてみて、\var{source}が有効なPythonコードの場合はコードオブジェクトを返します。このような場合、コードオブジェクトのファイル名属性は、デフォルトで\code{'<input>'}である\var{filename}でしょう。\var{source}が有効なPythonコードでは\emph{ない}が、有効なPythonコードの接頭語である場合には、\code{None}を返します。

\var{source}に問題がある場合は、例外を発生させます。無効なPython構文がある場合は、\exception{SyntaxError}を発生させます。また、無効なリテラルがある場合は、\exception{OverflowError}または\exception{ValueError}を発生させます。

\var{symbol}引数は\var{source}が文としてコンパイルされるか(\code{'single'}、デフォルト)、または式としてコンパイルされたかどうかを決定します(\code{'eval'})。他のどんな値も\exception{ValueError}を発生させる原因となります。

\strong{警告:}
ソースの終わりに達する前に、成功した結果をもってパーサは構文解析を止めることが(できそうではなく)できます。このような場合、後ろに続く記号はエラーとならずに無視されます。例えば、改行が後ろに付くバックスラッシュには不定のゴミが付いているかもしれません。パーサのAPIがより良くなればすぐに、これは修正されるでしょう。
\end{funcdesc}

\begin{classdesc}{Compile}{}
このクラスのインスタンスは組み込み関数\function{compile()}とシグネチャが一致する\method{__call__()}メソッドを持っていますが、インスタンスが\module{__future__}文を含むプログラムテキストをコンパイルする場合は、インスタンスは有効なその文とともに続くすべてのプログラムテキストを'覚えていて'コンパイルするという違いがあります。
\end{classdesc}

\begin{classdesc}{CommandCompiler}{}
このクラスのインスタンスは\function{compile_command()}とシグネチャが一致する\method{__call__()}メソッドを持っています。インスタンスが\code{__future__}文を含むプログラムテキストをコンパイルする場合に、インスタンスは有効なその文とともにそれに続くすべてのプログラムテキストを'覚えていて'コンパイルするという違いがあります。
\end{classdesc}

バージョン間の互換性についての注意: \class{Compile}と\class{CommandCompiler}はPython 2.2で導入されました。2.2のfuture-tracking機能を有効にするだけでなく、2.1とPythonのより以前のバージョンとの互換性も保ちたい場合は、次のようにかくことができます

\begin{verbatim}
try:
    from codeop import CommandCompiler
    compile_command = CommandCompiler()
    del CommandCompiler
except ImportError:
    from codeop import compile_command
\end{verbatim}

これは影響の小さい変更ですが、あなたのプログラムにおそらく望まれないグローバル状態を導入します。または、次のように書くこともできます:

\begin{verbatim}
try:
    from codeop import CommandCompiler
except ImportError:
    def CommandCompiler():
        from codeop import compile_command
        return compile_command
\end{verbatim}

そして、新たなコンパイラオブジェクトが必要となるたびに\code{CommandCompiler}を呼び出します。
