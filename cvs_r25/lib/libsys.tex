\section{\module{sys} ---
         システムパラメータと関数}

\declaremodule{builtin}{sys}
\modulesynopsis{システムパラメータと関数へのアクセス}

このモジュールでは、インタープリタで使用・管理している変数や、インタープ
リタの動作に深く関連する関数を定義しています。このモジュールは常に利用可能で
す。

\begin{datadesc}{argv}
  Pythonスクリプトに渡されたコマンドライン引数のリスト。\code{argv[0]}は
  スクリプトの名前となりますが、フルパス名かどうかは、オペレーティングシ
  ステムによって異なります。コマンドライン引数に\programopt{-c}を付けて
  Pythonを起動した場合、\code{argv[0]}は文字列\code{'-c'}となります。引
  数なしでPythonを起動した場合、\code{argv}は長さ0のリストになります。
\end{datadesc}

\begin{datadesc}{byteorder}
  プラットフォームのバイト順を示します。ビッグエンディアン(最上位バ
  イトが先頭)のプラットフォームでは\code{'big'}、リトルエンディアン(最下
  位バイトが先頭)では\code{'little'}となります。
  \versionadded{2.0}
\end{datadesc}

\begin{datadesc}{subversion}
3つ組 (repo, branch, version) で Python インタプリタの Subversion
情報を表します。
\var{repo} はリポジトリの名前で、\code{'CPython'}。
\var{branch} は \code{'trunk'}、\code{'branches/name'} または
\code{'tags/name'} のいずれかの形式の文字列です。
\var{version} はもしインタプリタが Subversion のチェックアウトから
ビルドされたものならば \code{svnversion} の出力であり、
リビジョン番号 (範囲) とローカルでの変更がある場合には最後に 'M' が付きます。
ツリーがエクスポートされたもの (または svnversion が取得できない) で、
branch がタグならば \code{Include/patchlevel.h} のリビジョンになります。
それ以外の場合には \code{None} です。
  \versionadded{2.5}
\end{datadesc}

\begin{datadesc}{builtin_module_names}
  コンパイル時にPythonインタープリタに組み込まれた、全てのモジュール名の
  タプル(この情報は、他の手段では取得することができません。
  \code{modules.keys()}は、インポートされたモジュールのみのリストを返し
  ます。)
\end{datadesc}

\begin{datadesc}{copyright}
  Pythonインタープリタの著作権を表示する文字列。
\end{datadesc}

\begin{funcdesc}{_current_frames}{}
  各スレッドの識別子を関数が呼ばれた時点のそのスレッドでアクティブになっ
  ている一番上のスタックフレームに結びつける辞書を返します。モジュー
  ル \refmodule{traceback} の関数を使えばそのように与えられたフレームの
  コールスタックを構築できます。

  この関数はデッドロックをデバッグするのに非常に有効です。デッドロック
  状態のスレッドの協調動作を必要としませんし、そういったスレッドのコー
  ルスタックはデッドロックである限り凍り付いたままです。デッドロックに
  ないスレッドのフレームについては、そのフレームを調べるコードを呼んだ
  時にはそのスレッドの現在の実行状況とは関係ないところを指し示している
  かもしれません。

  この関数は外部に見せない特別な目的でのみ使われるべきです。
  \versionadded{2.5}
\end{funcdesc}

\begin{datadesc}{dllhandle}
  Python DLLのハンドルを示す整数。
  利用可能: Windows
\end{datadesc}

\begin{funcdesc}{displayhook}{\var{value}}
  \var{value}が\code{None}以外の場合、\code{value}を\code{sys.stdout}に
  出力して\code{__builtin__._}に保存します。
  
  \code{sys.displayhook}は、Pythonの対話セッションで入力された式が評価さ
  れたときに呼び出されます。対話セッションの出力をカスタマイズする場合、
  \code{sys.displayhook}に引数の数が一つの関数を指定します。
\end{funcdesc}

\begin{funcdesc}{excepthook}{\var{type}, \var{value}, \var{traceback}}
  指定したトレースバックと例外を\code{sys.stderr}に出力します。
  
  例外が発生し、その例外が捕捉されない場合、インタープリタは例外クラス・
  例外インスタンス・トレースバックオブジェクトを引数として
  \code{sys.excepthook}を呼び出します。対話セッション中に発生した場合は
  プロンプトに戻る直前に呼び出され、Pythonプログラムの実行中に発生した場合
  はプログラムの終了直前に呼び出されます。このトップレベルでの例外情報出
  力処理をカスタマイズする場合、\code{sys.excepthook}に引数の数が三つの
  関数を指定します。
\end{funcdesc}

\begin{datadesc}{__displayhook__}
\dataline{__excepthook__}
  それぞれ、起動時の\code{displayhook}と\code{excepthook}の値を保存して
  います。この値は、\code{displayhook}と\code{excepthook}に不正なオブジ
  ェクトが指定された場合に、元の値に復旧するために使用します。
\end{datadesc}

\begin{funcdesc}{exc_info}{}
  この関数は、現在処理中の例外を示す三つの値のタプルを返します。この値
  は、現在のスレッド・現在のスタックフレームのものです。現在のスタックフ
  レームが例外処理中でない場合、例外処理中のスタックフレームが見つかるま
  で次々とその呼び出し元スタックフレームを調べます。ここで、``例外処理中
  ''とは``except節を実行中、または実行した''フレームを指します。どのスタ
  ックフレームでも、最後に処理した例外の情報のみを参照することができま
  す。

  スタック上で例外が発生していない場合、三つの\code{None}のタプルを返し
  ます。例外が発生している場合、\code{(\var{type}, \var{value},
  \var{traceback})}を返します。\var{type}は、処理中の例外の型を示します
  (クラスオブジェクト)。\var{value}は、例外パラメータ
  (例外に\dfn{関連する値}または\keyword{raise}の第二引数。\var{type}が
  クラスオブジェクトの場合は常にクラスインスタンス)です。\var{traceback}
  は、トレースバックオブジェクトで、例外が発生した時点でのコールスタック
  をカプセル化したオブジェクトです(リファレンスマニュアル参照)。
  \obindex{traceback}

  \function{exc_clear()}が呼び出されると、現在のスレッドで他の例外が発生するか、
  又は別の例外を処理中のフレームに実行スタックが復帰するまで、
  \function{exc_info()}は三つの\code{None}を返します。
  
  \warning{例外処理中に戻り値の\var{traceback}をローカル変数に代入すると
  循環参照が発生し、関数内のローカル変数やトレースバックが参照している全
  てのオブジェクトは解放されなくなります。特にトレースバック情報が必要で
  はなければ\code{exctype, value = sys.exc_info()[:2]}のように例外型と例
  外オブジェクトのみを取得するようにして下さい。もしトレースバックが必要
  な場合には、処理終了後にdeleteして下さい。このdeleteは、\keyword{try}
  ... \keyword{finally} ...で行うと良いでしょう。}
  \note{Python 2.2 以降では、ガベージコレクションが有効であればこのような
  到達不能オブジェクトは自動的に削除されます。しかし、循環参照を作らない
  ようにしたほうが効率的です。}
\end{funcdesc}

\begin{funcdesc}{exc_clear}{}
  この関数は、現在のスレッドで処理中、又は最後に発生した例外の情報を全てクリア
  します。この関数を呼び出すと、現在のスレッドで他の例外が発生するか、
  又は別の例外を処理中のフレームに実行スタックが復帰するまで、
  \function{exc_info()}は三つの\code{None}を返します。

  この関数が必要となることは滅多にありません。ロギングやエラー処理などで最後に
  発生したエラーの報告を行う場合などに使用します。また、リソースを解放して
  オブジェクトの終了処理を起動するために使用することもできますが、オブジェクト
  が実際にされるかどうかは保障の限りではありません。
\versionadded{2.3}
\end{funcdesc}


\begin{datadesc}{exc_type}
\dataline{exc_value}
\dataline{exc_traceback}
\deprecated {1.5}
            {\function{exc_info()}を使用してください}
  これらの変数はグローバル変数なのでスレッド毎の情報を示すことができませ
  ん。この為、マルチスレッドなプログラムでは安全に参照することはできませ
  ん。例外処理中でない場合、\code{exc_type}の値は\code{None}となり、
  \code{exc_value}と\code{exc_traceback}は未定義となります。
\end{datadesc}

\begin{datadesc}{exec_prefix}

  Pythonのプラットフォーム依存なファイルがインストールされているディレク
  トリ名(サイト固有)。デフォルトでは、この値は\code{'/usr/local'}です
  が、ビルド時に\program{configure}の\longprogramopt{exec-prefix}引数で
  指定することができます。全ての設定ファイル(\file{pyconfig.h}など)は
  \code{exec_prefix + '/lib/python\var{version}/config'}に、共有ライブラ
  リは\code{exec_prefix + '/lib/python\var{version}/lib-dynload'}にイン
  ストールされます(但し\var{version}は\code{version[:3]})。
  
\end{datadesc}

\begin{datadesc}{executable}
  Pythonインタープリタの実行ファイルの名前を示す文字列。このような名前が
  意味を持つシステムでは利用可能。
\end{datadesc}

\begin{funcdesc}{exit}{\optional{arg}}
  Pythonを終了します。\function{exit()}は\exception{SystemExit}を送出す
  るので、\keyword{try}ステートメントの\keyword{finally}節に終了処理を記
  述したり、上位レベルで例外を捕捉してexit処理を中断したりすることができ
  ます。オプション引数\var{arg}には、終了ステータスとして整数(デフォルト
  は0）または整数以外の型のオブジェクトを指定することができます。整数を
  指定した場合、シェル等は0は``正常終了''、0以外の整数を``異常終了''とし
  て扱います。多くのシステムでは、有効な終了ステータスは0-127で、これ以
  外の値を返した場合の動作は未定義です。システムによっては特定の終了コー
  ドに個別の意味を持たせている場合がありますが、このような定義は僅かしか
  ありません。\UNIX{}プログラムでは文法エラーの場合には2を、それ以外のエ
  ラーならば1を返します。\var{arg}に\var{None}を指定した場合は、数値の0
  を指定した場合と同じです。それ以外のオブジェクトを指定すると、そのオブ
  ジェクトが\code{sys.stderr}に出力され、終了コードをして1を返します。エ
  ラー発生時には\code{sys.exit("エラーメッセージ")}と書くと、簡単にプロ
  グラムを終了することができます。
\end{funcdesc}

\begin{datadesc}{exitfunc}
  この値はモジュールに存在しませんが、ユーザプログラムでプログラム終了時
  に呼び出される終了処理関数として、引数の数が0の関数を設定することがで
  きます。この関数は、インタープリタ終了時に呼び出されます。
  \code{exitfunc}に指定することができる終了処理関数は一つだけですので、
  複数のクリーンアップ処理が必要な場合は\refmodule{atexit}モジュールを使
  用してください。
  \note{プログラムがシグナルでkillされた場合，Python内部で致命的なエラーが
    発生した場合，\code{os._exit()}が呼び出された場合には，
    終了処理関数は呼び出されません。}
  \deprecated{2.4}{\refmodule{atexit} を使ってください．}
\end{datadesc}

\begin{funcdesc}{getcheckinterval}{}
インタプリタの ``チェックインターバル (check interval)'' を
返します; \function{setcheckinterval()} を参照してください。
  \versionadded{2.3}
\end{funcdesc}

\begin{funcdesc}{getdefaultencoding}{}
  現在のUnicode処理のデフォルトエンコーディング名を返します。
  \versionadded{2.0}
\end{funcdesc}

\begin{funcdesc}{getdlopenflags}{}
  \cfunction{dlopen()}で指定されるフラグを返します。このフラグは
  \refmodule{dl}と\module{DLFCN}で定義されています。
  
  利用可能: \UNIX.
  \versionadded{2.2}
\end{funcdesc}

\begin{funcdesc}{getfilesystemencoding}{}
  Unicodeファイル名をシステムのファイル名に変換する際に使用する
  エンコード名を返します。システムのデフォルトエンコーディングを使用する
  場合には\code{None}を返します。
\begin{itemize}
\item Windows 9xでは、エンコーディングは ``mbcs'' となります。
\item OS Xでは、エンコーディングは ``utf-8'' となります。
\item \UNIX では、エンコーディングは\code{nl_langinfo(CODESET)}が返す
      ユーザの設定となります。
      \code{nl_langinfo(CODESET)}が失敗すると\constant{None}を返します。
\item Windows NT+では、Unicodeをファイル名として使用できるので
      変換の必要はありません。\function{getfilesystemencoding()} は
      \code{'mbcs'}
      を返しますが、これはある Unicode 文字列をバイト文字列に明示的に変換
      して、ファイル名として使うと同じファイルを指すようにしたい場合に、
      アプリケーションが使わねばならないエンコーディングです。
\end{itemize}
  \versionadded{2.3}
\end{funcdesc}

\begin{funcdesc}{getrefcount}{object}
  \var{object}の参照数を返します。\var{object}は(一時的に)
  \function{getrefcount()}からも参照されるため、参照数は予想される数
  よりも1多くなります。
\end{funcdesc}

\begin{funcdesc}{getrecursionlimit}{}
  現在の最大再帰数を返します。最大再帰数は、Pythonインタープリタスタック
  の最大の深さです。この制限はPythonプログラムが無限に再帰し、Cスタック
  がオーバーフローしてクラッシュすることを防止するために設けられていま
  す。この値は\function{setrecursionlimit()}で指定することができます。
\end{funcdesc}

\begin{funcdesc}{_getframe}{\optional{depth}}
  コールスタックからフレームオブジェクトを取得します。オプション引数
  \var{depth}を指定すると、スタックのトップから\var{depth}だけ下のフレー
  ムオブジェクトを取得します。\var{depth}がコールスタックよりも深けれ
  ば、\exception{ValueError}が発生します。\var{depth}のデフォルト値は0
  で、この場合はコールスタックのトップのフレームを返します。
  
  この関数は、内部的な、特殊な用途にのみ利用することができます。
\end{funcdesc}

\begin{funcdesc}{getwindowsversion}{}
  実行中のWindowsのバージョンを示す、以下の値のタプルを返します：
  \var{major}, \var{minor}, \var{build}, \var{platform}, \var{text}。
  \var{text}は文字列、それ以外の値は整数です。

  \var{platform}は、以下の値となります:
  \begin{tableii}{l|l}{constant}{Constant}{Platform}
    \lineii{0 (VER_PLATFORM_WIN32s)}       {Win32s on Windows 3.1}
    \lineii{1 (VER_PLATFORM_WIN32_WINDOWS)}{Windows 95/98/ME}
    \lineii{2 (VER_PLATFORM_WIN32_NT)}     {Windows NT/2000/XP}
    \lineii{3 (VER_PLATFORM_WIN32_CE)}     {Windows CE}
  \end{tableii}
  
  この関数は、Win32 \function{GetVersionEx()}関数を呼び出します。詳細は
  マイクロソフトのドキュメントを参照してください。

  利用可能: Windows.
  \versionadded{2.3}
\end{funcdesc}

\begin{datadesc}{hexversion}
  整数にエンコードされたバージョン番号。この値は新バージョン(正規リリー
  ス以外であっても)ごとにかならず増加します。例えば、Python 1.5.2以降で
  のみ動作するプログラムでは、以下のようなチェックを行います。

\begin{verbatim}
if sys.hexversion >= 0x010502F0:
    # use some advanced feature
    ...
else:
    # use an alternative implementation or warn the user
    ...
\end{verbatim}
  \samp{hexversion}は\function{hex()}で16進数に変換しなければ値の意味が
  わかりません。より読みやすいバージョン番号が必要な場合には
  \code{version_info}を使用してください。
  \versionadded{1.5.2}
\end{datadesc}

\begin{datadesc}{last_type}
\dataline{last_value}
\dataline{last_traceback}
  通常は定義されておらず、捕捉されない例外が発生してインタープリタがエ
  ラーメッセージとトレースバックを出力した場合にのみ設定されます。この値
  は、対話セッション中にエラーが発生したとき、デバッグモジュールをロード
  (例:\samp{import pdb; pdb.pm()}など。詳細は\ref{debugger}を参照)して発
  生したエラーを調査する場合に利用します。デバッガをロードすると、プログ
  ラムを再実行せずに情報を取得することができます。

  変数の意味は、上の\function{exc_info()}の戻り値と同じです。対話セッシ
  ョンを実行するスレッドは常に一つだけなので、\code{exc_type}のようにス
  レッドに関する問題は発生しません。
\end{datadesc}

\begin{datadesc}{maxint}
  Pythonの整数型でサポートされる、最大の整数。この値は最低でも2**31-1で
  す。最大の負数は\code{-maxint-1}となります。正負の最大数が非対称です
  が、これは2の補数計算を行うためです。
\end{datadesc}

\begin{datadesc}{maxunicode}
  Unicode文字の最大のコードポイントを示す整数。この値は、オプション設定
  でUnicode文字の保存形式としてUSC-2とUCS-4のいずれを指定したかによって
  異なります。
\end{datadesc}

\begin{datadesc}{modules}
  ロード済みモジュールのモジュール名とモジュールオブジェクトの辞書。強制
  的にモジュールを再読み込みする場合などに使用します。この辞書からモジ
  ュールを削除するのは、\function{reload()}\bifuncindex{reload}の呼び出
  しと等価では\emph{ありません}。
\end{datadesc}

\begin{datadesc}{path}
\indexiii{module}{search}{path}
  モジュールを検索するパスを示す文字列のリスト。\envvar{PYTHONPATH}環境
  変数と、インストール時に指定したデフォルトパスで初期化されます。
  
  開始時に初期化された後、リストの先頭(\code{path[0]})にはPythonインター
  プリタを起動するために指定したスクリプトのディレクトリが挿入されます。
  スクリプトのディレクトリがない(インタープリタで対話セッションで起
  動された時や、スクリプトを標準入力から読み込む場合など)場合、
  \code{path[0]}には空文字列となり、Pythonはカレントディレクトリからモジ
  ュールの検索を開始します。スクリプトディレクトリは、
  \envvar{PYTHONPATH}で指定したディレクトリの\emph{前}に挿入されますので
  注意が必要です。

  必要に応じて、プログラム内で自由に変更することができます。

  \versionchanged[Unicode 文字列が無視されなくなりました]{2.3}
\end{datadesc}

\begin{datadesc}{platform}
  プラットフォームを識別する文字列(例: \code{'sunos5'}, \code{'linux1'}
  等)。\code{path}にプラットフォーム別のサブディレクトリを追加する場
  合などに利用します。
\end{datadesc}

\begin{datadesc}{prefix}
  サイト固有の、プラットフォームに依存しないファイルを格納するディレクト
  リを示す文字列。デフォルトでは\code{'/usr/local'}になります。この値は
  ビルド時に\program{configure}スクリプトの\longprogramopt{prefix}引数で
  指定する事ができます。Pythonライブラリの主要部分は
  \code{prefix + '/lib/python\var{version}'}にインストールされ、プラット
  フォーム非依存なヘッダファイル(\file{pyconfig.h}以外)は
  \code{prefix + '/include/python\var{version}'}に格納されます
  (但し\var{version}は\code{version[:3]})。
\end{datadesc}

\begin{datadesc}{ps1}
\dataline{ps2}
\index{interpreter prompts}
\index{prompts, interpreter}
  インタープリタの一次プロンプト、二次プロンプトを指定する文字列。対話
  モードで実行中のみ定義され、初期値は\code{'>>>~'}と
  \code{'...~'}です。文字列以外のオブジェクトを指定した場合、インタープ
  リタが対話コマンドを読み込むごとにオブジェクトの\function{str()}を評価
  します。この機能は、動的に変化するプロンプトを実装する場合に利用しま
  す。
\end{datadesc}

\begin{funcdesc}{setcheckinterval}{interval}
  インタープリタの``チェック間隔''を示す整数値を指定します。この値はスレ
  ッドスイッチやシグナルハンドラのチェックを行う周期を決定します。デフォ
  ルト値は\code{100}で、この場合 100 の仮想命令を実行するとチェックを行いま
  す。この値を大きくすればスレッドを利用するプログラムのパフォーマンスが
  向上します。この値が\code{<=0}以下の場合、全ての仮想命令を実行するたび
  にチェックを行い、レスポンス速度と最大になりますがオーバヘッドもまた最
  大となります。
\end{funcdesc}

\begin{funcdesc}{setdefaultencoding}{name}
  現在のUnicode処理のデフォルトエンコーディング名を設定します。
  \var{name}に一致するエンコーディングが見つからない場合、
  \exception{LookupError}が発生します。この関数は、\refmodule{site}モジ
  ュールの実装が、\module{sitecustomize}モジュールから使用するためだけに
  定義されています。\refmodule{site}から呼び出された後、この関数は
  \module{sys}から削除されます。
  \versionadded{2.0}
\end{funcdesc}

\begin{funcdesc}{setdlopenflags}{n}
  インタープリタが拡張モジュールをロードする時、\cfunction{dlopen()}で使
  用するフラグを設定します。\code{sys.setdlopenflags(0)}とすれば、モジ
  ュールインポート時にシンボルの遅延解決を行う事ができます。シンボルを拡
  張モジュール間で共有する場合には、
  \code{sys.setdlopenflags(dl.RTLD_NOW | dl.RTLD_GLOBAL)}と指定します。
  フラグの定義名は\refmodule{dl}か\module{DLFCN}で定義されています。
  \module{DLFCN}が存在しない場合、\program{h2py}スクリプトを使って
  \file{/usr/include/dlfcn.h}から生成することができます。

  利用可能: \UNIX.
  \versionadded{2.2}
\end{funcdesc}

\begin{funcdesc}{setprofile}{profilefunc}
  システムのプロファイル関数を登録します。プロファイル関数は、
Pythonのソースコードプロファイルを行う関数で、Pythonで記述する
ことができます。\index{profiler} 詳細は\ref{profile}を参照してく
  ださい。プロファイル関数はトレース関数(\function{settrace()}参照)と
  似ていますが、ソース行が実行されるごとに呼び出されるのではなく、関数の
  呼出しと復帰時のみ呼び出されます(例外が発生している場合でも、復帰時の
  イベントは発生します)。プロファイル関数はスレッド毎に設定することがで
  きますが、プロファイラはスレッド間のコンテキスト切り替えを検出すること
  はできません。従って、マルチスレッド環境でのプロファイルはあまり意味が
  ありません。\function{setprofile}は常に\code{None}を返します。
\end{funcdesc}

\begin{funcdesc}{setrecursionlimit}{limit}
  Pythonインタープリタの、スタックの最大の深さを\var{limit}に設定しま
  す。この制限はPythonプログラムが無限に再帰し、Cスタックがオーバーフ
  ローしてクラッシュすることを防止するために設けられています。
  
  \var{limit}の最大値はプラットフォームによって異なります。深い再帰処理
  が必要な場合にはプラットフォームがサポートしている範囲内でより大きな値
  を指定することができますが、この値が大きすぎればクラッシュするので注意
  が必要です。
\end{funcdesc}

\begin{funcdesc}{settrace}{tracefunc}
  システムのトレース関数を登録します。トレース関数
  はPythonのソースデバッガを実装するために使用することができます。
  \ref{debugger-hooks}の``How It Works,''を参照してください。
  \index{debugger} トレース関数はスレッド毎に設定することができますの
  で、デバッグを行う全てのスレッドで\function{settrace()}を呼び出し、ト
  レース関数を登録してください。
  \note{\function{settrace()} 関数は，デバッガ，プロファイラ，
    カバレッジツール等で使うためだけのものです．
    この関数の挙動は言語定義よりも実装プラットフォームの分野の問題で，
    全ての Python 実装で利用できるとは限りません．}
\end{funcdesc}

\begin{funcdesc}{settscdump}{on_flag}
  \var{on_flag} が真の場合，Pentium タイムスタンプカウンタを使った
  VM 計測結果のダンプ出力を有効にします．\var{on_flag} をオフにすると
  ダンプ出力を無効化します．この関数は Python を
  \longprogramopt{with-tsc} つきでコンパイルしたときにのみ利用できます．
  ダンプの内容を理解したければ， Python ソースコード中の
  \file{Python/ceval.c} を読んでください．
  \versionadded{2.4}
\end{funcdesc}

\begin{datadesc}{stdin}
\dataline{stdout}
\dataline{stderr}
  インタープリタの標準入力・標準出力・標準エラー出力に対応するファイルオ
  ブジェクト。\code{stdin}はスクリプトの読み込みを除く全ての入力処理で使
  用され、\function{input()}\bifuncindex{input}や\function{raw_input()}
  \bifuncindex{raw_input}も\code{stdin}から読み込みます。\code{stdout}
  は、\keyword{print}や式の評価結果、\function{input()}・
  \function{raw_input()}のプロンプトの出力先となります。インタープリタの
  プロンプトは(ほとんど)\code{stderr}に出力されます。\code{stdout}と
  \code{stderr}は必ずしも組み込みのファイルオブジェクトである必要はな
  く、\method{write()}メソッドを持つオブジェクトであれば使用することがで
  きます。\code{stdout}と\code{stderr}を別のオブジェクトに置き換えても、
  \function{os.popen()}・\function{os.system()} ・\refmodule{os}の
  \function{exec*()}などから起動されたプロセスが使用する標準I/Oストリー
  ムは変更されません。
\end{datadesc}

\begin{datadesc}{__stdin__}
\dataline{__stdout__}
\dataline{__stderr__}
  それぞれ起動時の\code{stdin}・\code{stderr}・\code{stdout}の値を保存し
  ます。終了処理時や、不正なオブジェクトが指定された場合に元の値に復旧す
  るために使用します。
\end{datadesc}

\begin{datadesc}{tracebacklimit}
  捕捉されない例外が発生した時、出力されるトレースバック情報の最大レベル
  数を指定する整数値(デフォルト値は\code{1000})。\code{0}以下の値が設定
  された場合、トレースバック情報は出力されず例外型と例外値のみが出力さ
  れます。
\end{datadesc}

\begin{datadesc}{version}
  Pythonインタープリタのバージョンとビルド番号・使用コンパイラなどの情
  報を示す文字列で、\code{'\var{バージョン}(\#\var{ビルド番号},
  \var{ビルド日付}, \var{ビルド時間})[\var{コンパイラ}]'}となります。先
  頭の三文字は、バージョンごとのインストール先ディレクトリ内を識別するた
  めに使用されます。例:

\begin{verbatim}
>>> import sys
>>> sys.version
'1.5.2 (#0 Apr 13 1999, 10:51:12) [MSC 32 bit (Intel)]'
\end{verbatim}
\end{datadesc}

\begin{datadesc}{api_version}
  使用中のインタープリタのC APIバージョン。Pythonと拡張モジュール間の不
  整合をデバッグする場合などに利用できます。\versionadded{2.3}
\end{datadesc}

\begin{datadesc}{version_info}
  バージョン番号を示す５つの値のタプル:\var{major}, \var{minor},
  \var{micro}, \var{releaselevel}, \var{serial} \var{releaselevel}以外は
  全て整数です。\var{releaselevel}の値は、\code{'alpha'}, \code{'beta'},
  \code{'candidate'}, or \code{'final'}の何れかです。Python 2.0の
  \code{version_info}は、\code{(2, 0, 0, 'final', 0)}となります。
  \versionadded{2.0}
\end{datadesc}

\begin{datadesc}{warnoptions}
  この値は、warnings framework内部のみ使用され、変更することはできませ
  ん。詳細は\refmodule{warnings}を参照してください。
\end{datadesc}

\begin{datadesc}{winver}
  Windowsプラットフォームで、レジストリのキーとなるバージョン番号。
  Python DLLの文字列リソース 1000に設定されています。通常、この値は
  \constant{version}の先頭三文字となります。この値は参照専用で、別の値を
  設定してもPythonが使用するレジストリキーを変更することはできません。
  利用可能: Windows.
\end{datadesc}
