\section{\module{fpectl} ---
         浮動小数点例外の制御}

\declaremodule{extension}{fpectl}
  \platform{Unix}
\moduleauthor{Lee Busby}{busby1@llnl.gov}
\sectionauthor{Lee Busby}{busby1@llnl.gov}
\modulesynopsis{Provide control for floating point exception handling.}
\modulesynopsis{浮動小数点例外処理の制御。}

ほとんどのコンピュータはいわゆるIEEE-754標準に準拠した浮動小数点演算\index{IEEE-754}を実行します。実際のどんなコンピュータでも、浮動小数点演算が普通の浮動小数点数では表せない結果になることがあります。例えば、次を試してください。

\begin{verbatim}
>>> import math
>>> math.exp(1000)
inf
>>> math.exp(1000) / math.exp(1000)
nan
\end{verbatim}

(上の例は多くのプラットホームで動作します。DEC Alphaは例外かもしれません。) "Inf"は"infinity(無限)"を意味するIEEE-754における特殊な非数値の値で、"nan"は"not a number(数ではない)"を意味します。ここで留意すべき点は、その計算を行うようにPythonに求めたときに非数値の結果以外に特別なことは何も起きないというです。事実、それはIEEE-754標準に規定されたデフォルトのふるまいで、それで良ければここで読むのを止めてください。

いくつかの環境では、誤った演算がなされたところで例外を発生し、処理を止めることがより良いでしょう。\module{fpectl}モジュールはそんな状況で使うためのものです。いくつかのハードウェア製造メーカーの浮動小数点ユニットを制御できるようにします。つまり、IEEE-754例外Division by Zero、OverflowあるいはInvalid Operationが起きたときはいつでも\constant{SIGFPE}が生成させるように、ユーザが切り替えられるようにします。あなたのpythonシステムを構成しているCコードの中へ挿入される一組のラッパーマクロと協力して、\constant{SIGFPE}は捕捉され、Python \exception{FloatingPointError}例外へ変換されます。

\module{fpectl}モジュールは次の関数を定義しています。また、所定の例外を発生します:

\begin{funcdesc}{turnon_sigfpe}{}
\constant{SIGFPE}を生成するように切り替え、適切なシグナルハンドラを設定します。
\end{funcdesc}

\begin{funcdesc}{turnoff_sigfpe}{}
浮動小数点例外のデフォルトの処理に再設定します。
\end{funcdesc}

\begin{excdesc}{FloatingPointError}
\function{turnon_sigfpe()}が実行された後に、IEEE-754例外であるDivision by Zero、OverflowまたはInvalid operationの一つを発生する浮動小数点演算は、次にこの標準Python例外を発生します。
\end{excdesc}


\subsection{例 \label{fpectl-example}}

以下の例は\module{fpectl}モジュールの使用を開始する方法とモジュールのテスト演算について示しています。

\begin{verbatim}
>>> import fpectl
>>> import fpetest
>>> fpectl.turnon_sigfpe()
>>> fpetest.test()
overflow        PASS
FloatingPointError: Overflow

div by 0        PASS
FloatingPointError: Division by zero
  [ more output from test elided ]
>>> import math
>>> math.exp(1000)
Traceback (most recent call last):
  File "<stdin>", line 1, in ?
FloatingPointError: in math_1
\end{verbatim}


\subsection{制限と他に考慮すべきこと}

特定のプロセッサをIEEE-754浮動小数点エラーを捕らえるように設定することは、現在アーキテクチャごとの基準に基づきカスタムコードを必要とします。あなたの特殊なハードウェアを制御するために\module{fpectl}を修正することもできます。

IEEE-754例外のPython例外への変換には、ラッパーマクロ\code{PyFPE_START_PROTECT}と\code{PyFPE_END_PROTECT}があなたのコードに適切な方法で挿入されていることが必要です。Python自身は\module{fpectl}モジュールをサポートするために修正されていますが、数値解析にとって興味ある多くの他のコードはそうではありません。

\module{fpectl}モジュールはスレッドセーフではありません。
