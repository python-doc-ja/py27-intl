\section{\module{gopherlib} ---
         gopher プロトコルのクライアント}

\declaremodule{standard}{gopherlib}
\modulesynopsis{gopher プロトコルのクライアント (ソケットを必要とします)。}

\deprecated{2.5}{ \code{gopher} プロトコルはあまり利用されていません。}

\indexii{Gopher}{protocol}

このモジュールでは、gopher プロトコルのクライアント側について最小限の
実装を提供しています。このモジュールは \refmodule{urllib} モジュールで
gopher プロトコルを使う URL を扱うために用いられます。

このモジュールでは以下の関数を定義しています:

\begin{funcdesc}{send_selector}{selector, host\optional{, port}}
\var{selector} 文字列を \var{host} および \var{port}
(標準の値は \code{70} です) の gopher サーバに送信します。
返信されたドキュメントデータを読み出すための、開かれた状態のファイル
オブジェクトを返します。
\end{funcdesc}

\begin{funcdesc}{send_query}{selector, query, host\optional{, port}}
\var{selector} 文字列、および \var{query} 文字列を \var{host} 
および \var{port} (標準の値は \code{70} です) の gopher サーバに送信します。
返信されたドキュメントデータを読み出すための、開かれた状態のファイル
オブジェクトを返します。
\end{funcdesc}

gopher サーバから返されるデータは任意の形式であり、セレクタ (selector)
文字列の最初の文字に依存するので注意してください。データがテキスト
 (セレクタの最初の文字が \samp{0}) の場合、各行は CRLF で終端
され、データ全体は \samp{.} 一個だけからなる行で終端されます。
\samp{..} で始まる行の先頭は \samp{.} に置き換えられます。
ディレクトリリスト (セレクタの最初の文字が \samp{1}) の場合にも、
同じプロトコルで転送されます。
