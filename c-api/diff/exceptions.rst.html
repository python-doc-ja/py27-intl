432+Recursion Control
433+=================
434+ 
435+These two functions provide a way to perform safe recursive calls at the C
436+level, both in the core and in extension modules.  They are needed if the
437+recursive code does not necessarily invoke Python code (which tracks its
438+recursion depth automatically).

440+.. cfunction:: int Py_EnterRecursiveCall(char *where)
441+ 
442+   Marks a point where a recursive C-level call is about to be performed.
443+ 
444+   If :const:`USE_STACKCHECK` is defined, this function checks if the the OS
445+   stack overflowed using :cfunc:`PyOS_CheckStack`.  In this is the case, it
446+   sets a :exc:`MemoryError` and returns a nonzero value.
447+ 
448+   The function then checks if the recursion limit is reached.  If this is the
449+   case, a :exc:`RuntimeError` is set and a nonzero value is returned.
450+   Otherwise, zero is returned.
451+ 
452+   *where* should be a string such as ``" in instance check"`` to be
453+   concatenated to the :exc:`RuntimeError` message caused by the recursion depth
454+   limit.
