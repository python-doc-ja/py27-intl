98+.. _2to3-fixers:
99+ 
100+Fixers
101+------
102+ 
103+Each step of transforming code is encapsulated in a fixer.  The command ``2to3
104+-l`` lists them.  As :ref:`documented above <2to3-using>`, each can be turned on
105+and off individually.  They are described here in more detail.
106+ 
107+ 
108+.. 2to3fixer:: apply
109+ 
110+   Removes usage of :func:`apply`.  For example ``apply(function, *args,
111+   **kwargs)`` is converted to ``function(*args, **kwargs)``.
112+ 
113+.. 2to3fixer:: basestring
114+ 
115+   Converts :class:`basestring` to :class:`str`.
116+ 
117+.. 2to3fixer:: buffer
118+ 
119+   Converts :class:`buffer` to :class:`memoryview`.  This fixer is optional
120+   because the :class:`memoryview` API is similar but not exactly the same as
121+   that of :class:`buffer`.
122+ 
123+.. 2to3fixer:: callable
124+ 
125+   Converts ``callable(x)`` to ``isinstance(x, collections.Callable)``, adding
126+   an import to :mod:`collections` if needed.
127+ 
128+.. 2to3fixer:: dict
129+ 
130+   Fixes dictionary iteration methods.  :meth:`dict.iteritems` is converted to
131+   :meth:`dict.items`, :meth:`dict.iterkeys` to :meth:`dict.keys`, and
132+   :meth:`dict.itervalues` to :meth:`dict.values`.  Similarly,
133+   :meth:`dict.viewitems`, :meth:`dict.viewkeys` and :meth:`dict.viewvalues` are
134+   converted respectively to :meth:`dict.items`, :meth:`dict.keys` and
135+   :meth:`dict.values`.  It also wraps existing usages of :meth:`dict.items`,
136+   :meth:`dict.keys`, and :meth:`dict.values` in a call to :class:`list`.
137+ 
138+.. 2to3fixer:: except
139+ 
140+   Converts ``except X, T`` to ``except X as T``.
141+ 
142+.. 2to3fixer:: exec
143+ 
144+   Converts the :keyword:`exec` statement to the :func:`exec` function.
145+ 
146+.. 2to3fixer:: execfile
147+ 
148+   Removes usage of :func:`execfile`.  The argument to :func:`execfile` is
149+   wrapped in calls to :func:`open`, :func:`compile`, and :func:`exec`.
150+ 
151+.. 2to3fixer:: exitfunc
152+ 
153+   Changes assignment of :attr:`sys.exitfunc` to use of the :mod:`atexit`
154+   module.
155+ 
156+.. 2to3fixer:: filter
157+ 
158+   Wraps :func:`filter` usage in a :class:`list` call.
159+ 
160+.. 2to3fixer:: funcattrs
161+ 
162+   Fixes function attributes that have been renamed.  For example,
163+   ``my_function.func_closure`` is converted to ``my_function.__closure__``.
164+ 
165+.. 2to3fixer:: future
166+ 
167+   Removes ``from __future__ import new_feature`` statements.
168+ 
169+.. 2to3fixer:: getcwdu
170+ 
171+   Renames :func:`os.getcwdu` to :func:`os.getcwd`.
172+ 
173+.. 2to3fixer:: has_key
174+ 
175+   Changes ``dict.has_key(key)`` to ``key in dict``.
176+ 
177+.. 2to3fixer:: idioms
178+ 
179+   This optional fixer performs several transformations that make Python code
180+   more idiomatic.  Type comparisons like ``type(x) is SomeClass`` and
181+   ``type(x) == SomeClass`` are converted to ``isinstance(x, SomeClass)``.
182+   ``while 1`` becomes ``while True``.  This fixer also tries to make use of
183+   :func:`sorted` in appropriate places.  For example, this block ::
184+ 
185+       L = list(some_iterable)
186+       L.sort()
187+ 
188+   is changed to ::
189+ 
190+      L = sorted(some_iterable)
191+ 
192+.. 2to3fixer:: import
193+ 
194+   Detects sibling imports and converts them to relative imports.
195+ 
196+.. 2to3fixer:: imports
197+ 
198+   Handles module renames in the standard library.
199+ 
200+.. 2to3fixer:: imports2
201+ 
202+   Handles other modules renames in the standard library.  It is separate from
203+   the :2to3fixer:`imports` fixer only because of technical limitations.
204+ 
205+.. 2to3fixer:: input
206+ 
207+   Converts ``input(prompt)`` to ``eval(input(prompt))``
208+ 
209+.. 2to3fixer:: intern
210+ 
211+   Converts :func:`intern` to :func:`sys.intern`.
212+ 
213+.. 2to3fixer:: isinstance
214+ 
215+   Fixes duplicate types in the second argument of :func:`isinstance`.  For
216+   example, ``isinstance(x, (int, int))`` is converted to ``isinstance(x,
217+   (int))``.
218+ 
219+.. 2to3fixer:: itertools_imports
220+ 
221+   Removes imports of :func:`itertools.ifilter`, :func:`itertools.izip`, and
222+   :func:`itertools.imap`.  Imports of :func:`itertools.ifilterfalse` are also
223+   changed to :func:`itertools.filterfalse`.
224+ 
225+.. 2to3fixer:: itertools
226+ 
227+   Changes usage of :func:`itertools.ifilter`, :func:`itertools.izip`, and
228+   :func:`itertools.imap` to their built-in equivalents.
229+   :func:`itertools.ifilterfalse` is changed to :func:`itertools.filterfalse`.
230+ 
231+.. 2to3fixer:: long
232+ 
233+   Strips the ``L`` prefix on long literals and renames :class:`long` to
234+   :class:`int`.
235+ 
236+.. 2to3fixer:: map
237+ 
238+   Wraps :func:`map` in a :class:`list` call.  It also changes ``map(None, x)``
239+   to ``list(x)``.  Using ``from future_builtins import map`` disables this
240+   fixer.
241+ 
242+.. 2to3fixer:: metaclass
243+ 
244+   Converts the old metaclass syntax (``__metaclass__ = Meta`` in the class
245+   body) to the new (``class X(metaclass=Meta)``).
246+ 
247+.. 2to3fixer:: methodattrs
248+ 
249+   Fixes old method attribute names.  For example, ``meth.im_func`` is converted
250+   to ``meth.__func__``.
251+ 
252+.. 2to3fixer:: ne
253+ 
254+   Converts the old not-equal syntax, ``<>``, to ``!=``.
255+ 
256+.. 2to3fixer:: next
257+ 
258+   Converts the use of iterator's :meth:`~iterator.next` methods to the
259+   :func:`next` function.  It also renames :meth:`next` methods to
260+   :meth:`~object.__next__`.
261+ 
262+.. 2to3fixer:: nonzero
263+ 
264+   Renames :meth:`~object.__nonzero__` to :meth:`~object.__bool__`.
265+ 
266+.. 2to3fixer:: numliterals
267+ 
268+   Converts octal literals into the new syntax.
269+ 
270+.. 2to3fixer:: paren
271+ 
272+   Add extra parenthesis where they are required in list comprehensions.  For
273+   example, ``[x for x in 1, 2]`` becomes ``[x for x in (1, 2)]``.
274+ 
275+.. 2to3fixer:: print
276+ 
277+   Converts the :keyword:`print` statement to the :func:`print` function.
278+
