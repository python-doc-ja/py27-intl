\section{\module{site} ---
         サイト固有の設定フック}

\declaremodule{standard}{site}
\modulesynopsis{サイト固有のモジュールを参照する標準の方法。}


\strong{このモジュールは初期化中に自動的にインポートされます。}
自動インポートはインタプリタの\programopt{-S}オプションで禁止できます。

このモジュールをインポートすることで、サイト固有のパスをモジュール検索
パスへ付け加えます。

\indexiii{module}{search}{path}

前部と後部からなる最大で四つまでのディレクトリを作成することから始めます。前部には、\code{sys.prefix}と\code{sys.exec_prefix}を使用します。空の前部は省略されます。
後部には、まず空文字列を使い、次に \file{lib/site-packages}(Windows) または 
\file{lib/python\shortversion/site-packages}、
そして \file{lib/site-python} (\UNIX{} と Macintosh)を使います。
別個の前部-後部の組み合わせのそれぞれに対して、それが存在するディレクトリを参照しているかどうかを調べ、もしそうならば\code{sys.path}へ追加します。そして、設定ファイルを新しく追加されたパスからも検索します。
\indexii{site-python}{directory}
\indexii{site-packages}{directory}

パス設定ファイルは\file{\var{package}.pth}という形式の名前をもつファイルで、上の4つのディレクトリのひとつにあります。その内容は\code{sys.path}に追加される追加項目(一行に一つ)です。存在しない項目は\code{sys.path}へは決して追加されませんが、項目が(ファイルではなく)ディレクトリを参照しているかどうかはチェックされません。項目が\code{sys.path}へ二回以上追加されることはありません。空行と\code{\#}で始まる行は読み飛ばされます。\code{import}で始まる行は実行されます。
\index{package}
\indexiii{path}{configuration}{file}

例えば、\code{sys.prefix}と\code{sys.exec_prefix}が\file{/usr/local}に設定されていると仮定します。そのときPython \version\ ライブラリは\file{/usr/local/lib/python\shortversion}にインストールされています(ここで、\code{sys.version}の最初の三文字だけがインストールパス名を作るために使われます)。ここにはサブディレクトリ\file{/usr/local/lib/python\shortversion/site-packages}があり、その中に三つのサブディレクトリ\file{foo}、\file{bar}および\file{spam}と二つのパス設定ファイル\file{foo.pth}と\file{bar.pth}をもつと仮定します。\file{foo.pth}には以下のものが記載されていると想定してください:

\begin{verbatim}
# foo package configuration

foo
bar
bletch
\end{verbatim}

また、\file{bar.pth}には:

\begin{verbatim}
# bar package configuration

bar
\end{verbatim}

が記載されているとします。そのとき、次のディレクトリが\code{sys.path}へこの順番んで追加されます:

\begin{verbatim}
/usr/local/lib/python2.3/site-packages/bar
/usr/local/lib/python2.3/site-packages/foo
\end{verbatim}

\file{bletch}は存在しないため省略されるということに注意してください。\file{bar}ディレクトリは\file{foo}ディレクトリの前に来ます。なぜなら、\file{bar.pth}がアルファベット順で\file{foo.pth}の前に来るからです。また、\file{spam}はどちらのパス設定ファイルにも記載されていないため、省略されます。

これらのパス操作の後に、\module{sitecustomize}\refmodindex{sitecustomize}という名前のモジュールをインポートしようします。そのモジュールは任意のサイト固有のカスタマイゼーションを行うことができます。\exception{ImportError}例外が発生してこのインポートに失敗した場合は、何も表示せずに無視されます。

いくつかの非\UNIX{}システムでは、\code{sys.prefix}と\code{sys.exec_prefix}は空で、パス操作は省略されます。しかし、\module{sitecustomize}\refmodindex{sitecustomize}のインポートはそのときでも試みられます。
